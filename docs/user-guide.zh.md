# apexcn-cli 用户手册

本手册面向通过 AI 工具使用 apexcn-cli 的 APEX 开发者。你无需记忆命令或研究参数；说明目标后，让 AI 助手完成检索、总结和整理。涉及发布、修改或删除时，始终先查看预览并明确确认。

## 1. 第一次使用

先只安装 CLI，不要在安装命令中提供 API key：

> 请帮我安装 apexcn-cli 和对应 skill。安装阶段不要请求、保存或验证 API key。安装完成后，请先确认实际运行的是刚安装的版本。

安装成功后，再单独配置认证：

> 我会在自己的 shell 中设置 APEXCN_API_KEY。请只让 apexcn-cli 保存这个环境变量名，不要读取或打印它的值；然后验证账号、板块列表和搜索能力。

如果你已经安装过，可以说：

> 请检查 apexcn-cli 是否已经可用，并确认当前登录的是哪个社区账号。不要输出完整 API key。

## 2. 搜索社区帖子

> 请用 apexcn-cli 搜索“APEX REST API”相关帖子，帮我总结前 5 条，并带上每条帖子的真实链接和原文链接。

> 请搜索“ORDS 认证失败”相关内容，把结果按“最可能有用、可以补充阅读、相关性一般”分组。

> 请搜索“APEX JSON_TABLE”的帖子，按从基础到进阶的顺序列出来。

## 3. 查看和总结某个帖子

> 请打开帖子 30549，帮我总结它主要讲了什么，列出关键步骤、注意事项和真实链接。

> 请读取这个帖子，并判断它是否适合解决我现在的问题：我在 APEX 调用 REST API 时拿不到返回 JSON。

## 4. 查账号和板块

> 请列出 APEX 中文社区可以发帖的板块，并告诉我每个板块适合发什么内容。

## 5. 让社区知识回答问题

如果想查看个人工作台能力，可以说：

> 请先检查服务端个人工作台能力矩阵，再查看我的通知、收件箱、社区规则和隐私政策。缺失能力必须明确说 unavailable，不要编造内容。

> 请用 apexcn-cli 根据社区内容回答：Oracle APEX 如何调用 REST API？请列出参考帖子链接。

> 请根据社区已有内容，整理一份 ORDS OAuth2 Bearer Token 排查清单。

## 6. 起草帖子

> 我想在社区提问：APEX 调用 REST API 返回 401。请先帮我搜索相关帖子，再帮我起草一篇清楚的问题描述。先不要发布，等我确认。

> 请根据下面的问题帮我写一篇社区求助帖，包含背景、已尝试方法、报错信息、期望结果和标签。写好后先给我看，不要直接发布。

## 7. 发布帖子

如果暂时不发布，可以先说：

> 请把刚才的问题草稿保存到当前 profile 的本地草稿清单。以后我说“列出我的草稿”时只显示这个 profile 的内容。

需要换 profile 或迁移到另一台机器时：

> 请导出当前 profile 的本地草稿，切换到目标 profile 后再导入。不要显示 API key，也不要覆盖同名草稿，除非我明确确认。

> 请把我确认后的内容发布到合适的板块。发布前先告诉我你准备选择哪个板块、标题是什么、正文是什么、标签是什么，等我确认后再发布。

> 请发布这篇帖子。发布成功后，把帖子真实链接发给我。

## 14. 整理成自己的知识笔记

> 请搜索“APEX REST API”相关帖子，把前 5 条整理成学习笔记。每条包含：适合谁看、主要内容、可操作步骤、真实链接、原文链接。

> 请把这些帖子整理成一份“APEX 调用 REST API 学习路线”，从基础到进阶排序。

也可以让 AI 建立离线资料包：

如果需要本地离线检索：

> 请为我的本地 collection 建立 BM25 索引，然后用 explain 模式查询 ORDS 认证失败。

## 15. 更换 API Key 或重新配置

> 请帮我重新配置 apexcn-cli 的 API key。我会提供新的 key。配置后请验证账号是否可用，不要输出完整 key。

也可以自己执行 `apexcn -apikey "你的真实 API KEY"`；普通字母数字 key 不加引号也可以。必须替换示例文字，且要注意命令可能进入 shell 历史。

## 16. 故障诊断

> apexcn-cli 好像不能用了。请先运行内置诊断，检查安装位置、登录状态、账号信息、板块列表和搜索能力，并告诉我是哪一步失败。

> AI 工具好像没有识别 apexcn-cli。请检查 skill 是否安装到当前 AI 工具能读取的位置。
