# apexcn-cli v0.80.0

`v0.80.0` 发布当前已经完成并通过本地质量门禁的组织治理、兼容性、凭据安全、诊断支持与生命周期能力。

## 新增能力

- 为 topic/reply 的 create、update、delete 操作提供显式 workflow policy。未配置操作默认拒绝；创建和更新至少需要一名审批者，删除至少需要两名不同审批者。
- workflow 审计事件使用 hash chain，并可验证保存的 JSON/NDJSON 审计记录；缺失、重排、篡改和额外事件都会失败关闭。
- `me capabilities` 识别 `0.8`、`0.7`、`0.6` 服务端能力窗口，支持 `--require-capability`，并拒绝未来、过旧、缺失或畸形 contract。
- `auth set-token --token-env` 支持从环境变量安全配置凭据；运行时按环境变量到本地文件的顺序回退，缺失时失败关闭，并避免输出秘密值。
- doctor snapshot 以 `0600` 权限写入，扩展敏感字段脱敏，并通过确定性 10,000-case redaction fuzz。
- 新增 macOS/Linux shell 与 Windows PowerShell 生命周期脚本，覆盖安装、升级、回滚、卸载、备份恢复和认证状态保留。
- 新增 issue routing matrix、只读 soak runner 和 v0.80 迁移说明。

## 当前资格边界

本版本发布的是当前已完成的 CLI 能力，不代表 roadmap 0.8 已完成。0.8 继续保持 `in_progress`，其 acceptance criteria 和 completion review 均未关闭。

- 新手独立验证仅完成 intake、候选安装、版本、help 和 commands 检查；用户暂停后没有继续产品场景，也没有产生 CLI finding。
- Windows 实机生命周期验证尚未执行。
- 七天只读 soak 尚未完成；暂停前仅有 4 次成功操作，不足以声明 99.5% 七天成功率。
- 因此不声明完整跨平台生命周期、七天 soak 或 0.8 独立验收已经通过。
- 0.5 继续保持 blocked/unreleased。
