# apexcn-cli v0.80.5

This patch prevents custom-path installer and lifecycle runs from overwriting an unrelated `apexcn` launcher found elsewhere on `PATH`.

## Changes

- Restricts shell-resolved launcher replacement to normal one-click installs that use the default install and bin directories.
- Applies the same containment rule to the PowerShell installer.
- Keeps the intended default-install behavior: a recognized older `apexcn-cli` launcher can still be refreshed so the current shell resolves the new release.
- Gives lifecycle tests an operation-specific isolated `PATH` instead of inheriting the developer machine's active launcher.
- Adds a regression that reproduces the external-launcher overwrite with custom paths and proves the external file remains byte-identical after installation.

## Verification

- The new containment regression failed against v0.80.4 behavior and passed after the fix.
- Focused older-launcher upgrade and POSIX lifecycle install/upgrade/rollback/uninstall tests passed.
- TypeScript build, full automated tests, release checks, source-layout checks, workflow checks, and roadmap checks passed.
