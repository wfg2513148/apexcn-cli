# apexcn-cli v0.80.3

This patch fixes community topic links returned by `apexcn ask` and consumed by Codex.

## Changes

- Converts relative APEX `card_link` and `threadUrl` values into absolute `https://oracleapex.cn/ords/f?p=...` topic URLs.
- Preserves the server-provided `cs` checksum so generated links open outside the current APEX session.
- Stops inventing the unsupported `https://oracleapex.cn/t/{id}` fallback when a reference has only a topic ID.
- Keeps external source links in `originalUrl` while exposing the community topic URL through `url` and `threadUrl`.
- Adds focused command tests and a natural-language Codex scenario for clickable source links.

## Verification

- Reproduced HTTP `404` for the old `/t/23722` fallback and HTTP `200` for the checksum-bearing APEX topic URL.
- Verified the locally built CLI against the live readonly `ask` endpoint and opened the first returned topic URL with HTTP `200`.
- TypeScript build and the full automated test suite passed.
- Release version, artifact, workflow, source-layout, and roadmap checks passed.

Milestone 0.8 remains in progress. Real-Windows lifecycle validation, fresh independent novice validation, and the seven-day readonly soak remain paused and are not claimed by this patch.
