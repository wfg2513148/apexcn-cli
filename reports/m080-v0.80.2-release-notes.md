# apexcn-cli v0.80.2

This patch adds the simplest API-key setup requested by users.

## Changes

- Added `apexcn -apikey "YOUR_API_KEY"` as a top-level authentication shortcut.
- Supports ordinary alphanumeric keys with or without shell quotes.
- Saves the key to the default `prod` profile at the production ORDS base URL and makes that profile active.
- Preserves `0600` config permissions, never echoes the key, and makes no community API request during configuration.
- Continues to reject blank, non-ASCII, whitespace-containing, and example-placeholder values.
- Keeps `auth set-token --token-env` and advanced multi-profile configuration unchanged.
- Updated installer guidance, README, quickstart, user guides, CLI manuals, migration notes, and the bundled agent skill.

## Security note

Passing a secret as a command-line argument may leave it in shell history or expose it briefly in a process listing. Users with higher security requirements should continue using `auth set-token --token-env`.
